# assets — Static Media for Docs/README

**Purpose:** Non-code binary/media assets referenced from documentation:
`logos/` (project logo/favicon), `screenshots/` (dashboard, Investigation
Workspace, Threat Timeline captures for the root README), `diagrams/`
(rendered architecture images, mirrored from `docs/diagrams/` for README
embedding convenience).

**Why it exists:** Keeps binary assets out of `docs/` and `core/`, in one
predictable location the README and GitHub repository description can link to.
